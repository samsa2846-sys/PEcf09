# Отчёт по практической работе: логирование RAG-ассистента

**Дисциплина:** промпт-инжиниринг (модуль про метрики и логирование).  
**Репозиторий для сдачи:** [github.com/samsa2846-sys/PEcf09](https://github.com/samsa2846-sys/PEcf09)

---

## 1. Цель работы

Реализовать **логирование взаимодействий** пользователя с RAG-ассистентом (запрос, ответ, источник, признак кеша, время ответа) с сохранением в **SQLite**, возможностью **агрегированной статистики** и **экспорта в CSV**, в духе методики урока ([пример проекта 5–7 с Chroma/OpenAI](https://github.com/MrGAN12009/5-7-new_version)).

Стек ассистента: **YandexGPT** + **FAISS** + **SQLite-кеш** ответов (аналогично по смыслу проектам с Yandex Cloud).

---

## 2. Архитектура

| Компонент | Файл | Назначение |
|-----------|------|------------|
| RAG-пайплайн | `rag_pipeline.py` | Кеш → FAISS → промпт → YandexGPT → сохранение в кеш |
| Векторы | `vector_store.py` | Чанки из `data/docs.txt`, эмбеддинги Yandex, индекс FAISS |
| LLM | `yandexgpt_client.py` | Запросы к YandexGPT / embeddings API |
| Кеш ответов | `cache.py` | SQLite, ключ по хешу нормализованного запроса |
| **Логи** | **`db_logger.py`** | Таблица `interactions`, методы `log_interaction`, `log_error`, `get_stats`, `export_csv` |
| Консоль | `app.py` | Тайминг `query()`, запись в лог, команды `stats`, `export`, `clear` |
| Telegram (опц.) | `telegram_bot.py` | Тот же пайплайн; в лог пишутся `user_id` / `username` из Telegram |

Поток данных: пользователь → `app.py` / бот → `RAGPipeline.query()` → при успехе/ошибке **`DatabaseLogger`** → файл **`logs.db`** (путь можно задать через `LOG_DB_PATH`).

---

## 3. Состав логируемых полей

- время (`created_at`);  
- текст запроса и ответа (с усечением длины в коде, чтобы не раздувать БД);  
- `source`: `cache` или `yandexgpt+faiss:…`;  
- `user_id`, `username` (в консоли обезличенно: `console` / `local`);  
- `from_cache` (0/1);  
- `response_time_ms`;  
- при сбое — поле `error`, `source=error`;  
- короткий `context_snippet` по найденным чанкам.

**Этика и ПДн:** в отчётные логи не заносятся паспортные данные, телефоны и т.п.; для Telegram рекомендуется не вводить чувствительные данные в чат.

## 4. Как воспроизвести (для проверки)

1. Python **3.11–3.13** (для Windows на **3.14** нет готовых колёс `faiss-cpu`).  
2. `copy .env.example .env` → заполнить **`YANDEX_API_KEY`**, **`YANDEX_FOLDER_ID`**.  
3. `.\setup_venv.ps1` или вручную: `py -3.11 -m venv venv`, `pip install -r requirements.txt`.  
4. `.\run_chat.ps1` или `python app.py`.  
5. Задать вопрос по теме из `data/docs.txt`; повторить вопрос — ответ из **кеша**.  
6. Команды: **`stats`** (в т.ч. метрики логов), **`export`** → `logs_export.csv`.

Проверка окружения без запуска диалога: `python preflight.py`.

## 5. Известные исправления в ходе работы

- При ответе **из кеша** контекст хранился как список **строк**, а UI и лог ожидали список **словарей** с полем `text` — в `rag_pipeline.py` добавлено **приведение формата** при возврате из кеша (и защита в `app.py` / `telegram_bot.py`).
